## 需求 2: 主页加入房间模块交互改为点击展开

**状态:** 已完成

**描述:** 加入房间模块的交互逻辑应与创建房间一致——初始显示为一个可点击的卡片按钮，点击后展开显示房间号和密码输入框。

**当前行为:** 加入房间始终显示输入框。

## 需求 3: 房间页面显示实时服务器延迟

**状态:** 已完成

**描述:** 在房间内页面 header 区域显示当前用户到信令服务器的实时延迟（ping/pong RTT），单位 ms。

**实现方案:**
- 在 `signalService` 中增加 PING 消息发送与 PONG 响应计时
- 在 RoomPage header 中展示延迟数值（如 `12ms`）
- 每隔几秒测量一次

---

## 需求 4: 调音台风格通道条组件

**状态:** 已完成

**描述:** 为房间内每位成员创建调音台/DAW 风格的通道条（Channel Strip），包含：
- 实时输入电平表（垂直 LED meter）
- 推子（Fader）— 垂直滑块控制音量
- Mute 按钮（M）
- Solo 按钮（S）

**布局参考:** 垂直排列，自上而下：名称 → 电平表 → 推子 → M/S 按钮。通道条横向排列，类似硬件调音台。

---

## 需求 5: 通道条电平表增加 dB 刻度

**状态:** 待实现

**描述:** 在通道条（ChannelStrip）的电平表旁边增加 dB 刻度线和数值标签，方便用户判断音量大小。

**细节:**
- 刻度点：0 dB（顶部，削波警告）、-6、-12、-18（标称电平参考线）、-24、-36、-48 dB
- 刻度线水平对齐电平表对应高度位置
- 削波区域（> -3 dB）使用红色高亮
- 数据来源为 SPA1 包头 `level` 字段（dBFS, -127~0）

---

## 需求 6: 推子支持鼠标滚轮控制

**状态:** 待实现

**描述:** 通道条的推子（Fader）增加鼠标滚轮控制，滚轮上下滑动可精细调整推子位置。

**细节:**
- 监听 `wheel` 事件，鼠标悬停在推子区域即可使用，无需先点击聚焦
- 灵敏度：每次滚轮 tick 调整 0.5 ~ 1 dB
- 按住 Shift 键时进入精细模式，每 tick 调整 0.1 dB
- 推子当前值以 dB 显示在推子下方
