## 需求 2: 主页加入房间模块交互改为点击展开

**状态:** 已完成

**描述:** 加入房间模块的交互逻辑应与创建房间一致——初始显示为一个可点击的卡片按钮，点击后展开显示房间号和密码输入框。

**当前行为:** 加入房间始终显示输入框。
**目标行为:** 先显示卡片 → 点击 → 展开输入区域 → 输入房间号/密码 → 点击加入/取消。

---

## 需求 3: 房间页面显示实时服务器延迟

**状态:** 已完成

**描述:** 在房间内页面 header 区域显示当前用户到信令服务器的实时延迟（ping/pong RTT），单位 ms。

**实现方案:**
- 在 `signalService` 中增加 PING 消息发送与 PONG 响应计时
- 在 RoomPage header 中展示延迟数值（如 `12ms`）
- 每隔几秒测量一次

---

## 需求 4: 调音台风格通道条组件

**状态:** 已完成

**描述:** 为房间内每位成员创建调音台/DAW 风格的通道条（Channel Strip），包含：
- 实时输入电平表（垂直 LED meter）
- 推子（Fader）— 垂直滑块控制音量
- Mute 按钮（M）
- Solo 按钮（S）

**布局参考:** 垂直排列，自上而下：名称 → 电平表 → 推子 → M/S 按钮。通道条横向排列，类似硬件调音台。
